# Health Check & Validation (Future)

## Goal

Provide quick checks to see how "Rosetta-compliant" a repo is and catch misconfigurations.

## Commands

### `rosetta validate`

- Checks:
  - `.ai/master-skill.md` exists.
  - `.ai/AGENT.md` and `.ai/task.md` exist.
  - `.ai/memory/PROJECT_MEMORY.md` and `AUTO_MEMORY.md` exist.
  - `.ai/logs/daily/` contains at least one file.
  - IDE wrappers present for configured IDEs.
  - Skills folders contain `SKILL.md`.

- Output: List of warnings and suggested fixes.

### `rosetta health`

- Computes a simple score (0–100) based on:
  - Presence of core files.
  - Consistency of references to `.ai/` in wrappers.
  - Existence of `skills/` with at least one SKILL.
- Example output: "Rosetta Health: 84/100 – missing AUTO_MEMORY.md and no skills detected."

## Implementation Notes

- Read-only; no file writes.
- Meant as a safety tool before CI or sharing a repo.

# Rosetta Roadmap – Summary & Priorities

## Phase 1 – Extensibility Foundations

1. **Plugin System – Discovery & Resolution**
   - Implement multi-source skills/presets (`templates/`, `.rosetta/`, `~/.rosetta/`).
   - Add `list-skills` and `list-presets` commands.
2. **Config-Driven Scaffolding**
   - Add `.rosetta.json` support.
   - Respect `defaultPreset`, `defaultIdes`, `autoContext`.
   - Implement `--non-interactive` mode.
3. **Company Skill Packs (basic)**: Allow `skillSources` in config.

## Phase 2 – Migration & Adoption

4. **Migration Wizard v1**
   - `rosetta migrate` interactive flow.
   - Create `.ai/master-skill.md` from existing context files.
   - Regenerate wrappers with `.ai/` references.
5. **Health & Validation**: `rosetta validate` and `rosetta health`.

## Phase 3 – Ecosystem & UX

6. **Post-Scaffold Hooks**: Support `postScaffoldHooks` shell commands.
7. **Profiles**: Allow switching between config profiles.
8. **Example Templates**: Add `examples/` directory.

## Open Questions

- How to distribute organization skill packs (git vs npm vs registry)?
- How much logic should `inferStarterSkills` perform?
- Where to draw the line between Rosetta and external tools?

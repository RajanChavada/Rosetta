# Config-Driven Scaffolding – Implementation

## Loading Config

- On CLI start, look for `.rosetta.json` in:
  - current directory
  - parent directories (optional)

- Provide helper:
  ```js
  function loadConfig() {
    // returns { config, sourcePath } or { config: {}, sourcePath: null }
  }
  ```

## Applying Config to Scaffold

In `scaffoldNew()`:

1. Call `loadConfig()`.
2. Determine preset: CLI arg → `config.defaultPreset` → interactive prompt.
3. Determine context:
   - If `config.autoContext.enabled` && `--non-interactive`, use config directly.
   - Else, merge: Config overrides as defaults, remaining fields via `gatherContext()`.
4. Determine IDEs: CLI arg → `config.defaultIdes` → interactive prompt.
5. Determine starter skills: `config.skills.alwaysInclude` plus `inferStarterSkills(context)`.

## Post-Scaffold Hooks

After scaffold completes:

- If `config.postScaffoldHooks` exists:
  - Run each shell command sequentially.
  - Log success/failure.
  - Later: support JS hooks in `.rosetta/hooks/post-scaffold.js`.

## Non-Interactive Mode

`rosetta scaffold --non-interactive`:

- Requires: `defaultPreset`, `autoContext.enabled = true`, `defaultIdes`.
- No inquirer prompts; fail if config incomplete.

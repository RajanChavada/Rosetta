# Plugin System – Discovery & Resolution

## Search Order

When Rosetta looks for skills/presets:

1. Project-specific:
   - `.rosetta/skills/`
   - `.rosetta/presets/`
2. Repo root:
   - `skills/` (existing project skills)
3. User-global:
   - `~/.rosetta/skills/`
   - `~/.rosetta/presets/`
4. Core:
   - `templates/skills/`
   - `templates/presets/`

Later entries are fallback only.

## Resolution Rules

- Skills are identified by **ID**:
  - `node-express-postgres`
  - `testing-full-pyramid`
- File naming convention:
  - `node-express-postgres.skill.md`
  - `testing-full-pyramid.skill.md`

Resolution:

1. Given an ID `X`, Rosetta looks for `X.skill.md` in each skill source path in order.
2. First match wins.

## Implementation Notes

- Add a `getSkillSources()` utility that returns an ordered list of directories.
- Add `resolveSkillTemplate(id)` that returns `{ path, source }` or `null`.
- Same pattern for presets: `company-agentic.preset.md` resolved from preset sources.

## CLI UX

- `rosetta list-skills`:
  - Shows skills found with origin: `core`, `project`, `global`.
- `rosetta list-presets`:
  - Shows available presets with origin.

This makes plugin discovery transparent to developers.

# Plugin System – Company Skill Packs

## Goal

Enable companies to bundle and version their own skills without forking Rosetta.

## Approach

Two modes:

1. **Local folder** in repo:
   - `company-skills/`
   - Referenced from `.rosetta.json`:
     ```json
     {
       "skillSources": ["company-skills"]
     }
     ```

2. **Git-based pack**:
   - Clone into `~/.rosetta/packs/<org>/<pack>/skills/`.
   - Configured via a simple install command (later roadmap).

## Example: `company-skills/` Layout

```text
company-skills/
  user-onboarding.skill.md
  payments-compliance.skill.md
  observability-playbook.skill.md
Each file follows the SKILL format (frontmatter + sections).

Scaffold Integration
inferStarterSkills(context) can detect domain tags (e.g., Financial) and suggest company skills like payments-compliance.

Scaffold flow:

After context gathering, show:

"Detected company skills: payments-compliance, observability-playbook. Create them under skills/?"

Future Enhancements
Add metadata in frontmatter:

tags: [financial, compliance]

requires: [PROJECT_MEMORY]

Allow filtering skills by tags during scaffold.
```

# Rosetta Plugin System – Overview

## Goal

Allow teams to extend Rosetta with their own skills and presets **without** modifying core code:
- Drop-in skills folders (local or git-sourced).
- Optional NPM-style packages later.
- Preserve Rosetta core as a thin, stable orchestrator.

## High-Level Concept

Rosetta loads skills and presets from multiple sources:

1. **Core** (bundled):
   - `templates/skills/` – default SKILL boilerplates.
   - `templates/presets/` – default master-skill presets.

2. **Project-local**:
   - `./skills/` – user-created skills.
   - `./.rosetta/skills/` – project-only skill packs.
   - `./.rosetta/presets/` – project-only presets.

3. **Organization / global**:
   - `~/.rosetta/skills/`
   - `~/.rosetta/presets/`

Rosetta discovers these folders at runtime and exposes them during scaffold and `new-skill`.

## User Stories

- As a **team**, I can drop `company-skills/` into the repo and have Rosetta suggest them automatically.
- As an **individual dev**, I can keep a personal `~/.rosetta/skills/` library used across all projects.
- As an **OSS maintainer**, I can ship special presets in `.rosetta/presets/` alongside my project.

# Migration Wizard – Detection & Strategy

## Detection

Reuse and extend `findExistingAgentFiles()`:

- Look for:
  - `CLAUDE.md`
  - `.cursorrules`
  - `.github/copilot-instructions.md`
  - `.windsurf/rules/`
  - `skills/`
  - `.agent/skills/`
  - `.ai/master-skill.md` (existing Rosetta setup)

## Strategies for Creating `.ai/master-skill.md`

If `.ai/master-skill.md` is missing:

1. **Single-source import**:
   - Ask: "Create .ai/master-skill.md from which source?"
   - Choices: each existing file.
   - Behavior: Prepend a comment `<!-- Generated by rosetta from CLAUDE.md -->` and copy content.

2. **Merge all (concatenate)**:
   - Merge selected files in order.
   - Insert headers: `<!-- Source: CLAUDE.md -->`.

3. **Skip**:
   - Allow user to keep master minimal and manually populate later.

## Preserving Original Files

- Never delete source files automatically.
- Always create `.bak` when overwriting wrappers.
- Document in migration summary what was changed and where backups live.

# Migration Wizard – IDE Wrapper Update

## Goal

After master spec exists under `.ai/`, ensure IDE wrappers:
- Continue to function.
- Explicitly reference `.ai/master-skill.md` and `.ai/memory/*`.
- Preserve any custom content where possible.

## Strategy

1. **Backup existing wrappers**: `CLAUDE.md.bak`, `.cursorrules.bak`, etc.
2. **Generate new wrappers from templates** (opt-in):
   - Ask: "Regenerate IDE wrappers from Rosetta templates?"
   - If yes: Use `ensureFromTemplate`.
   - If no: Optionally append a block pointing to `.ai/`.
3. **Integrate Memory Block**: Ensure standard "Agent Memory & Logging Workflow" is present.
